Introdução

Bem vindo e bem vinda à Consultoria!

Este material tem o objetivo de te auxiliar na configuração e preparo do seu ambiente para o desenvolvimento dos projetos.

Softwares que utilizaremos:

  • GitBash – terminal de comandos e versionamento git
  • Bizagi – para desenho de BPMN
  • Eclipse com plugin Activiti – para desenho da programação de fluxos BPMN
  • NetBeans – para programação em Java
  • JDK – para programação em Java
  • NodeJS - desenvolvimento
  • VisualStudio Code –IDE
  • Postman ou Insomnia – testar endpoints e protocolos http GET/POST/etc
  • MySQL Workbench – Banco de Dados
  • Maven – compilação dos projetos em Java. Baixe o bin.

Instale-os em seu dispositivo e tenha sempre a última versão estável disponível.

2º Passo – Criando as variáveis de ambiente

Para que os projetos funcionem corretamente, é necessário configurar variáveis de ambiente no seu dispositivo. Caso esteja usando Windows, busque em Iniciar por “variáveis” e selecione:


Na janela que aparecer, clique em “Variáveis de Ambiente...”


Nas variáveis do sistema, criaremos as variáveis “JAVA_HOME” e “MAVEN_HOME”


Se ainda não existirem em seu sistema, clique em .

Caso já existam, configure em .

As variáveis devem ser:

  • Nome: JAVA_HOME
  • Local da variável: Caminho em sua máquina onde está instalado o JDK.
  • Ex.: C:\Program Files\Java\jdk-12.0.2

  • Nome: MAVEN_HOME
  • Local da variável: Caminho em sua máquina onde está instalado o Maven.
  • Ex.: C:\Program Files\apache-maven-3.6.3-bin\apache-maven-3.6.3

Assim que configuradas as variáveis JAVA e MAVEN, selecione a variável Path e clique em .

Na nova janela, clique em . Iremos adicionar %JAVA_HOME%\bin e %MAVEN_HOME%\bin

3º Passo – corrigindo compilação do projeto core

Navegue até o clone do projeto core-matrix-energia e então para o diretório “MatrixCore”. Aqui execute o seguinte comando pelo terminal:

mvn install:install-file -Dfile=./sqljdbc42.jar -DgroupId=com.microsoft.sqlserver -DartifactId=sqljdbc42 -Dversion=4.2 -Dpackaging=jar

Caso receba o erro “mvn command not found” a variável do sistema não está configurada corretamente. Feche o terminal, verifique as configurações dos passos anteriores e tente novamente.

4º Passo – Corrigindo o ambiente de teste no NetBeans

Abra o projeto MatrixCore no NetBeans e, com o botão direito do mouse, abra suas propriedades.

Vá até actions e, em debug project, corrija a propriedade “-Dspring.profiles.active=teste” para “-dev”

Conteúdo completo de Set Properties:

spring-boot.run.jvmArguments=-Xdebug -Xrunjdwp:transport=dt_socket,server=n,address=${jpda.address} -noverify -XX:TieredStopAtLevel=1 -Dspring.profiles.active=dev
spring-boot.run.mainClass=com.core.matrix.MatrixApplication
Env.SPRING_OUTPUT_ANSI_ENABLED=always
jpda.listen=true
